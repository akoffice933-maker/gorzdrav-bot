## 1. Общая информация о проекте

### 1.1. Описание продукта

Горздрав Бот — Telegram-бот для жителей Санкт-Петербурга, позволяющий записаться к врачу через портал «Горздрав СПб», а также получить ИИ-помощь по медицинским вопросам.

**Ключевые возможности существующего бота:**
- 🩺 Запись к врачу через API Горздрава (выбор района, поликлиники, врача, времени)
- 🔬 Расшифровка анализов — OCR фотографии + ИИ-анализ (Qwen via OpenRouter)
- 📋 Анализ симптомов и рекомендации по обследованиям
- 💊 Ответы на медицинские вопросы через языковую модель
- 📅 Автоматические напоминания о записи за 3 часа
- 🔐 Подготовка к авторизации через ЕСИА (Госуслуги)

### 1.2. Цель сайта

Создать публичный веб-сайт, который:
- Представляет продукт (лендинг) для конечных пользователей — жителей СПб
- Служит посадочной страницей для перехода в Telegram-бот
- Отражает доверие и безопасность (медицинская тематика, 152-ФЗ, ЕСИА)
- Содержит документацию и FAQ по работе с ботом
- Обеспечивает OAuth2-callback endpoint для будущей авторизации через ЕСИА

### 1.3. Целевая аудитория

| Сегмент | Описание | Ключевая потребность |
|---|---|---|
| Пациенты СПб | Жители, записывающиеся через Горздрав | Быстрая запись к врачу без очередей |
| Разработчики | Изучают код/API, потенциальные контрибьюторы | Документация, GitHub |
| Медицинские учреждения | Поликлиники, использующие Горздрав | Партнёрство, виджет записи |

## 2. Состав сайта

### 2.1. Структура страниц

| № | Страница / раздел | Назначение |
|---|---|---|
| 1 | `/` (Главная / Лендинг) | Hero, возможности бота, CTA «Открыть в Telegram» |
| 2 | `/features` | Детальное описание функций с иллюстрациями |
| 3 | `/faq` | Вопросы и ответы, инструкция по первому запуску |
| 4 | `/privacy` | Политика конфиденциальности (152-ФЗ) |
| 5 | `/esia/callback` | OAuth2-редирект (технический эндпоинт ЕСИА) |
| 6 | `/docs` | Документация API (опционально, для разработчиков) |

### 2.2. Главная страница — детальная структура

#### Hero-секция
- Заголовок: «Запишитесь к врачу в несколько кликов»
- Подзаголовок с кратким описанием бота
- Кнопка CTA: «Открыть бот в Telegram» (`t.me/...`)
- Анимированная иллюстрация — скриншот/мокап интерфейса бота
- Бейдж «Работает для СПб | Горздрав API»

#### Блок преимуществ (Features)
- 4–6 карточек: запись к врачу, ИИ-анализ, расшифровка анализов, напоминания, безопасность
- Каждая карточка: иконка + заголовок + 2 строки описания

#### Блок «Как это работает» (How It Works)
- Пошаговая схема: `/start → выбор функции → результат` (3–4 шага)
- Визуализация в виде горизонтального timeline или accordion

#### Блок безопасности
- Перечень мер: 152-ФЗ, шифрование, rate limiting, валидация, ЕСИА
- Акцент на конфиденциальность медицинских данных

#### Блок технологий
- Стек: Python / aiogram / SQLAlchemy / Qwen (OpenRouter) / EasyOCR
- Ссылка на GitHub репозиторий

#### FAQ (краткий блок)
- 3–5 наиболее частых вопросов с ответами в accordion-стиле

#### Footer
- Ссылки: GitHub, Telegram-бот, политика конфиденциальности
- Копирайт, контакты (Issue в GitHub)

## 3. Функциональные требования

### 3.1. Ключевые функции

| ID | Функция | Описание |
|---|---|---|
| F-01 | CTA-кнопка «Открыть бот» | Ссылка на Telegram-бот. Кнопка закреплена (sticky) на мобильных устройствах |
| F-02 | ЕСИА OAuth2 Callback | Эндпоинт `/esia/callback` принимает `code` и `state`, передаёт в бота через webhook или хранилище |
| F-03 | Форма обратной связи | Email-форма или ссылка на GitHub Issues |
| F-04 | Счётчик / метрика | Yandex.Metrika или аналог (с учётом 152-ФЗ) |
| F-05 | Переключатель языка | RU / EN (опционально, v2) |
| F-06 | Swagger / ReDoc | Документация REST API на `/docs` (опционально) |

### 3.2. ЕСИА OAuth2 Callback — технические требования

- `GET /esia/callback?code=...&state=...` — принять параметры
- Верифицировать `state` (защита от CSRF)
- Передать `code` в бэкенд Telegram-бота (через Redis/БД или webhook)
- Отобразить пользователю страницу: «Авторизация прошла успешно, вернитесь в Telegram»
- Поддерживать HTTPS (обязательно для регистрации в ЕСИА)

## 4. Нефункциональные требования

### 4.1. Производительность
- Lighthouse Performance Score ≥ 90
- LCP (Largest Contentful Paint) ≤ 2.5 сек
- FID (First Input Delay) ≤ 100 мс
- CLS (Cumulative Layout Shift) ≤ 0.1

### 4.2. Адаптивность
- Полная адаптация: Mobile (320–480 px), Tablet (768–1024 px), Desktop (1280+ px)
- Sticky CTA-кнопка на мобильных
- Touch-friendly элементы (min 44×44 px)

### 4.3. SEO
- Семантическая HTML-разметка (h1–h6, article, section, nav)
- Open Graph и Twitter Card мета-теги
- `robots.txt`, `sitemap.xml`
- Ключевые запросы: «записаться к врачу Санкт-Петербург», «Горздрав СПб бот»

### 4.4. Безопасность
- HTTPS (TLS 1.2+), SSL-сертификат
- Content Security Policy (CSP) заголовок
- HSTS, X-Content-Type-Options, X-Frame-Options
- Политика конфиденциальности по требованиям 152-ФЗ
- Минимальный сбор персональных данных

### 4.5. Доступность
- WCAG 2.1 уровень AA
- Alt-тексты для всех изображений
- Клавиатурная навигация
- Контрастность текста ≥ 4.5:1

## 5. Дизайн и UX

### 5.1. Стиль и тональность

Медицинский сервис требует сочетания доверия, чистоты и технологичности. Референсы: Gosuslugi.ru (доверие, государственный стиль), современные медицинские стартапы (чистота, иконографика).

### 5.2. Цветовая палитра

| Роль | HEX | Применение |
|---|---|---|
| Основной (primary) | `#1A6B8A` | Кнопки, заголовки, акценты |
| Вспомогательный | `#2D9CDB` | Hover-состояния, ссылки |
| Зелёный (успех) | `#2D6A4F` | Иконки функций, статусы |
| Фоновый | `#F4F9FC` | Фон страницы |
| Тёмный текст | `#1A1A2E` | Основной текст |
| Серый (вторичный) | `#6B7280` | Подписи, caption-текст |

### 5.3. Типографика
- Основной шрифт: **Inter** (Google Fonts) — нейтральный, читаемый
- Размеры: h1 48–56px, h2 32–36px, body 16px, small 14px
- Межстрочный интервал: 1.6 для основного текста

### 5.4. Иконографика и иллюстрации
- Lucide Icons или Heroicons (open source)
- Мокапы интерфейса бота — скриншоты Telegram или Figma-иллюстрации
- Медицинские иллюстрации — только из royalty-free источников (unDraw, Storyset)

## 6. Технический стек

### 6.1. Рекомендуемые варианты

| Слой | Вариант A (рекомендован) | Вариант B (упрощённый) |
|---|---|---|
| Frontend | Next.js 14 (App Router) | Vanilla HTML + Tailwind CSS |
| Стилизация | Tailwind CSS | Tailwind CSS / Bootstrap 5 |
| Backend | FastAPI (Python) — уже в стеке | Flask/FastAPI минимальный |
| ЕСИА Callback | FastAPI endpoint | FastAPI endpoint |
| Хостинг | VPS (ishosting.io) / Vercel + FastAPI | VPS, один контейнер |
| SSL | Let's Encrypt + Nginx | Let's Encrypt + Nginx |
| CI/CD | GitHub Actions | GitHub Actions |

> **Рекомендован Вариант A**: Next.js обеспечивает SSR/SSG, хорошо индексируется, а FastAPI органично вписывается в уже существующий Python-стек бота.

### 6.2. Инфраструктура
- **Nginx** — обратный прокси, SSL-терминация, статика
- **Docker Compose** — объединение сервисов сайта и бота
- **PostgreSQL** — опционально для хранения ЕСИА state
- **Redis** — хранение OAuth2 state tokens с TTL

## 7. Интеграции

### 7.1. Telegram-бот
- Ссылка `t.me/{bot_username}` — основная CTA
- Webhook от ЕСИА-callback к боту через внутренний HTTP-запрос или Redis pub/sub

### 7.2. ЕСИА (Госуслуги)
- Регистрация `redirect_uri`: `https://your-domain.ru/esia/callback`
- Подпись JWT с RSA-ключом (уже запланировано в боте)
- Хранение `state` с TTL в Redis для защиты от CSRF

### 7.3. Аналитика
- **Yandex.Metrika** — соответствует 152-ФЗ лучше, чем Google Analytics
- Опционально: Plausible Analytics (self-hosted, GDPR-friendly)

### 7.4. GitHub
- Блок «Open Source» со ссылкой на репозиторий
- Отображение актуальной версии/тега через GitHub API

## 11. Вне скоупа (Out of Scope)

Следующее **не входит** в данную версию и планируется в v2:
- Личный кабинет пользователя (история записей, анализов)
- Inline-виджет записи к врачу прямо на сайте
- Полная реализация ЕСИА JWT-авторизации (только инфраструктура callback)
- Мультиязычность (RU/EN)
- Админ-панель для статистики бота
- Мобильное приложение
