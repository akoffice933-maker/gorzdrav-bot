# 🏥 Медицинский Telegram-бот с ИИ

Бот для записи к врачу через портал «Горздрав СПб» с ИИ-анализом медицинских анализов и симптомов.

## 🌐 Веб-сайт проекта

Проект включает полноценный веб-сайт с информацией о боте, функциями, FAQ и политикой конфиденциальности. Сайт готов к развертыванию через GitHub Pages.

**Доступные страницы:**
- **Главная** (`index.html`) — описание бота, ключевые возможности
- **Функции** (`features.html`) — детальное описание всех функций
- **FAQ** (`faq.html`) — ответы на часто задаваемые вопросы
- **Политика конфиденциальности** (`privacy.html`) — правила обработки данных

**Ссылка на сайт:** [https://akoffice933-maker.github.io/gorzdrav-bot/](https://akoffice933-maker.github.io/gorzdrav-bot/)

## Функциональность

- 🩺 **Запись к врачу** — через API Горздрава с выбором района, поликлиники, врача и времени
- 🔬 **Расшифровка анализов** — OCR фото + ИИ-анализ результатов с отклонениями от нормы
- 💊 **Вопросы к ИИ** — ответы на медицинские вопросы через Qwen (OpenRouter)
- 📋 **Анализ симптомов** — рекомендации по анализам на основе симптомов
- 📅 **Напоминания** — автоматические напоминания о записи за 3 часа
- 🔐 **Авторизация ЕСИА** — подготовка для входа через Госуслуги
- 🌐 **Веб-сайт** — информационный сайт о проекте

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

> **Важно:** При первом запуске `easyocr` скачает модели для русского и английского языков (~2 ГБ).

### 2. Настройка конфигурации

Скопируйте `.env.example` в `.env` и укажите значения:

```bash
cp .env.example .env
```

Откройте `.env` и укажите:

```ini
BOT_TOKEN=ваш_токен_от_BotFather
OPENROUTER_API_KEY=ваш_ключ_от_openrouter.ai
ENCRYPTION_KEY=сгенерируйте_ключ_для_шифрования
```

Генерация ключа шифрования:
```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

### 3. Запуск бота

```bash
python bot.py
```

## Развертывание веб-сайта на GitHub Pages

Веб-сайт уже настроен для автоматического развертывания через GitHub Pages. Для активации:

1. Перейдите в настройки репозитория на GitHub → **Pages**
2. В разделе **Source** выберите ветку `main` и папку `/website`
3. Нажмите **Save**

Сайт будет доступен по адресу: `https://<ваш-username>.github.io/gorzdrav-bot/`

**Альтернативно** можно использовать GitHub Actions для автоматического деплоя. В проекте уже есть файл `DEPLOYMENT.md` в папке `website/` с детальными инструкциями.

## Получение API-ключей

### Telegram Bot Token
1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot` и следуйте инструкциям
3. Скопируйте полученный токен в `.env`

### OpenRouter API Key
1. Зарегистрируйтесь на [openrouter.ai](https://openrouter.ai)
2. Перейдите в раздел API Keys
3. Создайте новый ключ и скопируйте в `.env`

### ЕСИА (опционально)
1. Зарегистрируйте приложение в [личном кабинете разработчика ЕСИА](https://esia.gosuslugi.ru)
2. Получите `client_id` и настройте `redirect_uri`
3. Сгенерируйте RSA-ключ для подписи JWT

## Безопасность

### Что реализовано

| Мера | Описание |
|------|----------|
| **Переменные окружения** | Все секреты в `.env`, не в коде |
| **Валидация входных данных** | Санитизация всех пользовательских вводов |
| **Prompt Injection защита** | Паттерны для обнаружения атак на ИИ |
| **Rate Limiting** | Защита от спама и DDoS |
| **Логирование** | Все действия логируются с ротацией файлов |
| **Обработка ошибок** | Try/except везде, retry для API |
| **OCR в отдельном потоке** | Не блокирует event loop |
| **HTML экранирование** | Защита от XSS в данных БД |

## Технические детали

### Модели базы данных

**User:**
- `tg_id` — Telegram ID пользователя
- `district_id`, `lpu_id` — выбранный район и поликлиника
- `doctor_id`, `doctor_name` — выбранный врач
- `appointment_time` — время записи
- `reminder_sent` — флаг отправки напоминания
- `esia_token` — токен авторизации ЕСИА

**AnalysisHistory:**
- `tg_id` — ID пользователя
- `analysis_type` — тип анализа (ocr/text)
- `raw_text` — исходный текст
- `ai_analysis` — результат ИИ-анализа

### Обработка ошибок

- Все API-запросы с таймаутами (15-20 сек)
- Автоматический retry при ошибках OpenRouter (exponential backoff)
- Graceful degradation при недоступности сервисов
- Дисклеймер в каждом ответе ИИ
- Логирование всех ошибок с трейсбэком

### Rate Limiting

| Действие | Лимит | Окно |
|----------|-------|------|
| Команда /start | 10 | 60 сек |
| Кнопки меню | 5 | 60 сек |
| Анализ симптомов | 3 | 60 сек |
| Вопрос к ИИ | 3 | 60 сек |
| Фото анализа | 3 | 60 сек |
